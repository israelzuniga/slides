Gradient Boosting¶

GB es una de las tcnicas ms populares en ML para ganar competencias de Kaggle sobre datos estructurados (y con el feature engineering adecuado), entrenar sobre datasets en los que no tenemos business expertise para descubrir variables con poder predictivo o solucionar casi cualquier problema de aprendiza supervisado y ganar dinero rpido.

Origen de Boosting¶

La idea fue articulada por Michael Kearns sobre un clasificador dbil (weaker learner*) que puede convertirse en eficiente sobre el tiempo:

... an efficient algorithm for converting relatively poor hypotheses into very good hypotheses Thoughts on Hypothesis Boosting, 1988.

Hypothesis boosting was the idea of filtering observations, leaving those observations that the weak learner can handle and focusing on developing new weak learns to handle the remaining difficult observations.

AdaBoosting¶

AdaBoost y otros algoritmos relacionados son propuestos como un marco de aprendizaje estadstico por Leo Breiman como ARCing Algorithms:

Arcing is an acronym for Adaptive Reweighting and Combining. Each step in an arcing algorithm consists of a weighted minimization followed by a recomputation of [the classifiers] and [weighted input]. Prediction Games and Arching Algorithms, 1999

AdaBoosting¶

Despus, es extendido en 2001 por Jerome H. Friedman y llamado Gradient Boosting Machine

Se establece el GB como un problema de optimizacin donde la funcin objetivo es minimizar la prdida del modelo al agregar clasificadores dbiles mediante gradient descent/descenso de gradiente. Este algoritmo describe un modelo aditivo por etapas:

Un nuevo clasificador dbil es agregado a la vez a una estructura de clasificadores que no se modifican.

Entonces cómo funciona Gradient Boosting?¶

Se involucran tres elementos:

  • Una funcin de prdida a ser optimizada
  • Un clasificador dbil que hace prediccciones
  • Un modelo aditivo que incrementa sus clasificadores para minimizar la funcin de prdida

Gradient Boosting¶

  • Weak learner: es un clasificador que sin importar la distribucin del dataset de entrenamiento, siempre ser mejor que una eleccin al azar, cuando intenta etiquetar ($\hat{Y}$) los datos de una observacin ($X$).

Gradient Boosting¶

  • Regularization: Mide la complejidad del modelo
  • Training Loss: Mide que tan bien se ajusta nuestro modelo a los datos

Gradient Boosting¶

  • Training loss functions: Mtodos para la evaluacin de que tan bien un algoritmo en particular est modelando las observaciones dadas. Si la prediccin $\hat{Y}$ se desva demasiado del resultado actual ${Y}$ la funcin de prdida otorgar un valor muy alto. Gradualmente, con la ayuda de una funcin de optimizacin, la funcin de prdida aprende a reducir el error en la prediccin)

Gradient Boosting¶

  • Modelo Aditivo: Es un modelo donde se aaden rboles a la vez, y los existentes no se modifican. Se usa gradient descent para minimizar la prdida cuando se agrega un nuevo arbol.

One way to produce a weighted combination of classifiers which optimizes [the cost] is by gradient descent in function space. Boosting Algorithms as Gradient Descent in Function Space, 1999. Llew Mason

Normalmente se usa el descenso de gradiente para minimizar parmetros, p. ej. el coeficiente un una ecuacin de regresin o los pesos de una NN. Despus de calcular el error o la prdida, los pesos internos se actualizan para minimizar ese error. En vez de parametros, los clasificadores dbiles (decision trees). Despus de calcular la prdidad, para calcular el descenso de gradiente, debemos aadir un arbol para minimizar la prdida (seguir el gradiente). Este approach es conocido como functional gradient descent.

Mejoras al Gradient Boosting¶

Dado que GB es un algoritmo agresivo, puede sobre-entrenar en el dataset. Usando funciones de regularizacin que penalizen a la implementacin del algoritmo:

  • Constantes para rboles
  • Contraccin (shrinkage/learning rate)
  • Random Sampling (Stochastic GB)
  • Penalized Gradient Boosting (Regularized Gradient Boosting: 1 / 2 regularization of weights )

Tambin hay dos tcnicas de regularizacin adicionales para evitar el overfitting. El primero es la contraccin (shrinkage), que agrega un factor de escala a los nuevos pesos despus de cada paso de refuerzo. Esto evita que algunos rboles individuales dominen el conjunto, dejando espacio para mejorar. La segunda tcnica es el submuestreo de caractersticas/features (feature subsampling), donde se muestrea una fraccin de las observaciones en cada paso, para aumentar la variedad de rboles construidos. El submuestreo de caractersticas tambin aumenta la velocidad de clculo, permitiendo que XGBoost se escale de manera eficiente.

\begin{equation}\label{eq:loss} \begin{split} \mathcal{L}(\phi) =& \sum_{i} l( \hat{y}_i, y_i ) + \sum_{k}\Omega( f_{k} ) \\ \mbox{ donde }& \Omega(f) = \gamma T + \frac{1}{2} \lambda \|w\|^2 \end{split} \end{equation}

L1 regularization of weights. L2 regularization of weights. The additional regularization term helps to smooth the final learnt weights to avoid over-fitting. Intuitively, the regularized objective will tend to select a model employing simple and predictive functions.

Tree Boosting¶

Regularized learning objective¶

Como en la mayora de modelos, minimizar la prdida (la diferencia entre la prediccin $\hat{Y}$ y la etiqueta ${Y}$ ) es la meta principal y queremos que el modelo sea lo ms exacto posible. Tambin es importante agregar un trmino de regularizacin ($\Omega$) para evitar el Overfitting/sobre-entrenamiento. Expresamos la funcin de objetivo de la siguiente forma

$$ \begin{equation}\label{eq:loss} \begin{split} \mathcal{L}(\phi) =& \sum_{i} l( \hat{y}_i, y_i ) + \sum_{k}\Omega( f_{k} ) \\ \end{split} \end{equation} $$

Here $l$ is a differentiable convex loss function that measures the difference between the prediction $\hat{y}_i$ and the target $y_i$. The second term $\Omega$ penalizes the complexity of the model (i.e., the regression tree functions). The additional regularization term helps to smooth the final learnt weights to avoid over-fitting. Intuitively, the regularized objective will tend to select a model employing simple and predictive functions. A similar regularization technique has been used in Regularized greedy forest~(RGF)~ model. Our objective and the corresponding learning algorithm is simpler than RGF and easier to parallelize. When the regularization parameter is set to zero, the objective falls back to the traditional gradient tree boosting.

Tree Boosting¶

Gradient tree boosting¶

Cuando entrenamos un modelo, es comn optimizarlo tomando el gradiente. Sin embargo, esto es difcil de realizar en un conjunto de rboles. En su lugar, usamos una estrategia aditiva y agregamos un rbol nuevo a la vez, de modo que cada rbol que agreguemos optimice el objetivo. Esto nos da una funcin de puntuacin que podemos usar para evaluar la calidad de una estructura de rbol.

En pocas palabras, $g_i$ y $h_i$ se ordenan en sus hojas apropiadas, se suman y luego se puntan.

Tree Boosting¶

Ahora que tenemos un mtodo para evaluar la calidad de la estructura de rbol, necesitamos aprender la estructura del rbol. Es imposible enumerar/completar esto en un solo paso para todos los rboles posibles, por lo que se optimiza un nivel de rbol a la vez. Comenzando con una divisin en una hoja, calculamos la ganancia, que viene dada por:

\begin{equation}\label{eq:gain} \mathcal{L}_{split} =\frac{1}{2} \left[\frac{(\sum_{i\in I_L} g_i)^2}{\sum_{i\in I_L} h_i + \lambda}+\frac{(\sum_{i\in I_R} g_i)^2}{\sum_{i\in I_R} h_i + \lambda} - \frac{(\sum_{i\in I} g_i)^2}{\sum_{i\in I} h_i + \lambda}\right] - \gamma \end{equation}

Donde $I_j=\{i|q(x_i)=j\}$ es el conjunto de instancias de la hoja $j$

XGBoost: eXtreme Gradient Boosting¶

XGBoost es una implementacin Open Source de Gradient Boosting Machines/Gradient Boosted Decision Trees creada por Tianqi Chen, en co-autora con Carlos Guestrin, (del grupo DMLC https://dmlc.github.io/people.html, tambin creadores de MXNet para Deep Learning). Siendo pensada desde el origen para entrenar con velocidad de cmputo y performance.

Escrita en C++, est disponible para los SO ms populares (Windows, OS X, Linux) con las siguientes interfaces:

  • Command Line Inferface (CLI)
  • C++
  • Python
  • R
  • Julia
  • JVM: Java (Hadoop), Scala (Spark)

XGBoost Features¶

Model Features¶

Soporta tres maneras (implementaciones de algoritmos) de Gradient Boosting:

  • Gradient Boosting/Gradient Boosting Machine (Learning Rate)
  • Stochastic Gradient Boosting (sub-sampling: row & column)
  • Regularized Gradient Boosting ($L^1$, $L^2$)

System Features¶

Uso de algoritmos y estructuras de datos optimizados para exprimir nuestro hardware:

  • Paralelizacin: Uso de todos los cores durante entrenamiento (construccin de rboles)
  • Cmputo Distribuido (HPC: OpenMP, Spark, Yarn, CUDA/NCCL, Dask, Dask+Kubernetes): Para entrenamiento de modelos en clusters de cmputo
  • Out-of-Core Computing: Para datasets que no caben en memoria RAM

Algorithm Features¶

  • Manejo automtico de valores faltantes (Sparsity-aware Split Finding Algorithm)
  • Continued Training: Entrenamiento incremental sobre modelos previos

Por qué usar XGBoost?¶

En resumen, usamos XGBoost por las siguientes razones:

  • Fcil de usar e instalar
  • Eficiencia
    • Rpido
    • Parelizalizacin sin dolor en una sola computadora o en cluster *
  • Exactitud
    • Buenos resultados para la mayora de datasets (Pruning, Gradient Optimization)
  • Personalizacin:
    • Paramtros
    • Cambios en la funcin Objetivo y evaluacin

* Sabemos lo que has llorado al implementar en tu lenguaje favorito:

  • Concurrency (Threading, Async IO/Cooperative Multitasking)
  • Parallelism (Multiprocessing).

Preparación de datos para GB¶

XGBoost solo considera como entrada valores nmericos, lo que significa que debemos "codificar" nuestro dataset mediante:

  • Label Encoding (LabelEncoder)
  • OneHot Encoding (OneHotEncoder)
  • Feature Scaling (StandardScaler

import numpy as np

# Mean absolute percentage error
def mean_absolute_percentage_error(estimator, X, y):
    y_pred = estimator.predict(X)
    y_true = y
    return np.mean(np.abs((y_true - y_pred) / y_true))
